# Lab 2 – Regular Expression to NFA

## Objective

To implement Thompson's Construction algorithm for converting regular expressions into Non-deterministic Finite Automatons (NFAs). Through systematic application of transformation rules for union, concatenation, Kleene star, and plus operators, students will construct an NFA that recognizes exactly the language defined by the given regular expression.

## Theoretical Background

Thompson's Construction, developed by Ken Thompson in 1968, provides a systematic method to convert any regular expression into an NFA that accepts the same language (Aho et al., 2006). The algorithm is central to the design of efficient lexical analyzers and pattern matching engines.

The formal definition: A regular expression over alphabet Σ is either: (i) ∅ (empty set), (ii) ε (empty string), (iii) a ∈ Σ, or (iv) generated by the operations: r₁ | r₂ (union), r₁·r₂ (concatenation), r* (Kleene closure), or r+ (positive closure).

Thompson's Construction recursively decomposes a regular expression into simpler sub-expressions, constructing elementary NFA fragments for each symbol, then combining fragments via operator-specific transformation rules. Key principles:
- Each regular expression corresponds to an NFA with a unique start state and a unique accept state
- No incoming edges to the accept state
- State transitions labeled with symbols or epsilon transitions
- Base case: Single symbol a creates a two-state NFA with a-labeled transition
- Union: Combine two fragments A and B by creating new start/accept states with epsilon transitions
- Concatenation: Connect fragment A's accept to fragment B's start via epsilon
- Kleene star: Add epsilon loop from accept back to start, plus epsilon bypass
- Plus: As Kleene star but without epsilon bypass (at least one repetition)

The resulting NFA may contain epsilon transitions (spontaneous moves), which can be eliminated via epsilon closure computation. The NFA can subsequently be converted to a DFA through the subset construction algorithm, achieving efficient pattern matching with deterministic behavior.

## Algorithm

**Thompson's Construction Algorithm:**

1. **Base Case**: For symbol s ∈ Σ
   - Create two new states p, q
   - Add transition p --s--> q
   - Return fragment(p, q)

2. **Union (r₁ | r₂)**:
   - Get NFA fragments F1 and F2 from recursive calls
   - Create new start state p_new and accept state q_new
   - Add epsilon transitions: p_new --ε--> F1.start, p_new --ε--> F2.start
   - Add epsilon transitions: F1.accept --ε--> q_new, F2.accept --ε--> q_new
   - Return fragment(p_new, q_new)

3. **Concatenation (r₁·r₂)**:
   - Get fragments F1 and F2
   - Add epsilon transition: F1.accept --ε--> F2.start
   - Return fragment(F1.start, F2.accept)

4. **Kleene Star (r*)**:
   - Get fragment F1
   - Create new start p_new and accept q_new
   - Add epsilon transitions: p_new --ε--> F1.start, p_new --ε--> q_new
   - Add epsilon transitions: F1.accept --ε--> F1.start (loop), F1.accept --ε--> q_new
   - Return fragment(p_new, q_new)

5. **Plus (r+)** (one or more):
   - Similar to Kleene star but without epsilon bypass from start to accept
   - Enforces at least one repetition

6. **Parse and construct** via recursive descent, respecting operator precedence

## Example Used

```
Regular Expression: (a|b)*abb
```

This regex matches any string over alphabet {a, b} that ends with the sequence "abb". The construction proceeds as:
1. Parse 'a' as symbol NFA
2. Parse 'b' as symbol NFA
3. Union to get (a|b) NFA
4. Apply Kleene star to get (a|b)* NFA
5. Parse 'a', 'b', 'b' as symbol NFAs
6. Concatenate sequentially to get abb NFA
7. Concatenate (a|b)* with abb to complete final NFA

## Program Structure

- **`nfa_table_init()`**: Initialize NFA storage structures. Time: O(1).
- **`allocate_state()`**: Generate unique state identifiers. Time: O(1).
- **`add_transition(from, symbol, to)`**: Insert edge in transition table. Time: O(1).
- **`thompson_symbol(symbol)`**: Create elementary NFA fragment. Time: O(1).
- **`thompson_union(f1, f2)`**: Combine fragments via union using epsilon. Time: O(1).
- **`thompson_concatenation(f1, f2)`**: Concatenate fragment sequences. Time: O(1).
- **`thompson_kleene_star(frag)`**: Apply Kleene closure operator. Time: O(1).
- **`thompson_plus(frag)`**: Apply positive closure operator. Time: O(1).
- **`parse_regex(regex, pos)`**: Recursive descent parser for regex. Time: O(n).
- **`build_nfa_from_regex(regex)`**: Main entry point for construction. Time: O(n).
- **`display_nfa_transitions(nfa)`**: Tabular transition output. Time: O(t).
- **`display_nfa_diagram(nfa)`**: State-centric diagram output. Time: O(s+t).

## Complexity Analysis

| Metric | Complexity |
|--------|-----------|
| Time Complexity | O(n + s) – where n is regex length, s is symbol count |
| Space Complexity | O(s) – where s is number of states (linear in symbols) |

Thompson's Construction processes each symbol and operator in the input regex exactly once, yielding linear time complexity. The resulting NFA contains at most 2n states for a regex of n symbols, giving linear space complexity. The algorithm is incremental and efficient for practical regular expressions.
